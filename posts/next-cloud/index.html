关键点

基于 Filecoin 的火热情况，我们积累了很多台计算机设备，但 Filecoin 迟迟没有上线，我们需要考虑怎么让这些机器运行起来，并获得一些收入。

CDN，OSS，VPN …… 我脑海里盘过了很多想法，但我依旧觉得很普通，我总觉得它应该有与众不同的形式，或者说创新是我的本能，我不太喜欢做一个已经重复制造一个已经存在的东西。

2019.3.27 夜晚，David 在加州公路上狂飙，我在副驾驶上昏昏欲睡，脑海里突然飘过 Serverless 的声音，我猛一惊醒，我们要发财了。

David 好奇追问，怎么了？我没有回复，只是回答，我需要仔细思考一下。

回溯

“如果 Filecoin 不能上线怎么办？”

“即使 Filecoin 上线了，失败了怎么办？”

“由于各种骗局，社会上至少有几十万台设备闲置着，我们得做点什么！可是怎么做呢？等 Filecoin 吗？“

我们一直陷在存储、挖矿的世界跳不出来。

能不能用它们来挖 Storj，挖其他 Token ？

不如做 CDN，或者 OSS 吧？

用上万，上千万，未来可能是过亿的设备，仅仅做 CDN、OSS，那只是个笑话吧！

我们似乎只是盯着这些设备的存储功能。

但它们是一个个完整的计算机啊！

之前在 YouTube 上看到有玩家将多个 Raspberry Pi 合并为一个性能更强的计算机，那么这些设备可不可以组成一个超级计算机呢？那样就能提供计算服务了！

什么样的计算服务？

对啊 …… 什么样的呢？Serverless 是一个选择。

想象一下，现在我们通过部署在阿里云上的代码来把成千上万的设备组成一个超级计算机，然后我们在把这些代码从阿里云放到这个超级服务器里运行，太酷了！

继续

所有的云平台最基础的三个元素：

  • 计算
  • 存储
  • 网络

Service Mesh

我们完全可以做一个众包的云平台啊！

想象一下，我们有这样一个平台，用户说，我需要一台 xx 的服务器，hmmm，我们找从计算容量中筛选符合条件的设备，并向它们的所有者发送消息，“Hi，xx，我们这里需要几台xx 的设备，租金是x， 你愿意参与吗？”

用户一看，hmm， 这个报价是可以的，接单。

接单后，指定选择出租哪些设备，然后这些设备就开始下载我们制作好的 Docker 镜像和配置文件。

配置文件包含了使用者的网络 ID（便于彼此发现节点）

我们也会启动一个中心服务点，用来拉红线，以及接收每个节点的 agent 发送过来的监控信息。

  • 时序数据库
  • Serverless
  • 监控
  • 处理时效性要求较高的计算服务
